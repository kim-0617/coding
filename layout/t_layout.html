페이지 레이아웃 정리

페이지 레이아웃을 배치하는 방법은 대표적으로 3가지가 있습니다.
쉽고 간단하게 핵심만 집어서 가보겠습니다.

다음의 태그를 대상으로 레이아웃을 적용해 보겠습니다.
<div id="wrap">
    <header id="header"></header>
    <nav id="nav"></nav>
    <aside id="aside"></aside>
    <section id="section"></section>
    <footer id="footer"></footer>
</div>

1. float을 이용한 Layout

float 속성을 사용해 박스를 왼쪽(left) 또는 오른쪽(right)으로 "부유"시키는 레이아웃 기법으로, 기본 문서 배치의 흐름에서 벗어나 화면 배치를 유연하게 할 수 있습니다.
float 속성을 이용해 레이아웃을 배치했으면 그 후에 오는 요소들은 clear:both 속성을 이용해서 레이아웃 배치를 해줍니다. 

 /* float */
#aside{
    float: left;
    height: 780px;
}
#section{
    float: left;
    height: 780px;
}
#footer{
    height: 100px;
    clear: both; 
}
/* 반응형 (mediaquery사용) */
@media(max-width : 1220px) { /* 노트북 화면 */
    #wrap {
        width: 96%;  /* 양쪽여백 */
    }
}

@media(max-width : 768px) { /* 태블릿 화면 */
    #wrap {
        width: 100%; /* 여백 없음 */
    }
}

@media(max-width : 480px) { /* 모바일 화면 */
    #aside {
        width: 100%;  /* 레이아웃이 한줄로 내려온다. */
        height: 300px;
    }

    #section {
        width: 100%;
        height: 480px;
    }
}

2. flex를 이용한 Layout

flex는 뷰포트나 요소의 크기가 불명확하거나 동적으로 변할 때에도 효율적으로 요소를 배치, 정렬, 분산할 수 있는 방법을 제공하는 CSS3의 새로운 레이아웃 방식입니다. flexbox의 장점을 한 마디로 표현하면 '복잡한 계산 없이 요소의 크기와 순서를 유연하게 배치할 수 있다'라는 것입니다. 정렬, 방향, 순서, 크기 등을 유연하게 조절할 수 있기 때문에 별도의 분기 처리를 줄일 수 있고, CSS만으로 다양한 레이아웃을 구현할 수 있습니다. 반응형 레이아웃을 구상할 때도 아주 편리합니다. 

 /* flex */
#wrap {  /* 부모 요소에 flex를 주면 자식 요소들은 알아서 정렬된다. */
    width: 1200px;
    display: flex;
    flex-wrap: wrap; /* flex-item 요소들이 내부로직에 의해 분할되어 배치 */
}
/* 반응형 (mediaquery사용) */
@media(max-width : 1220px) { /* 노트북 화면 */
    #wrap {
        width: 96%; /* 양쪽 여백 */
    }
}

@media(max-width : 768px) { /* 태블릿 화면 */
    #wrap {
        width: 100%;  /* 여백 없음 */
    }
}

@media(max-width : 480px) { /* 모바일 화면 */
    #aside {
        width: 100%;   /* 레이아웃이 한줄로 내려온다. */
        height: 300px;
    }

    #section {
        width: 100%;
        height: 480px;
    }
}

3. grid를 이용한 Layout

CSS 그리드 레이아웃은 웹페이지를 위한 2차원 레이아웃 시스템 입니다. 그리드는 수평선과 수직선으로 이루어진 집합체로, 디자인 요소를 정렬할 수 있는 대상 패턴을 생성합니다. 이 디자인은 페이지에서 페이지로 이동할 때 요소가 널뛰거나 너비가 바뀌지 않는 디자인 생성에 도움을 주어 웹 사이트의 일관성을 높여줍니다. 

 /* grid */
#wrap {
    display: grid;  /* grid선언 */   
    grid-template-areas: /* grid레이아웃의 영역을 지정한다 */
        "header header"  /* row = 4, col = 2 */
        "nav nav"
        "aside section"
        "footer footer"
    ;
    grid-template-columns: 30% 70%; /* 열너비를 지정해준다. (col = 2기 때문에 2개만 지정) */
    grid-template-rows: 100px 100px 780px 100px; /* 각각의 행높이 지정 */ 
}

#header {
    grid-area: header; /* grid-template-areas에서 정의한 이름들의 영역을 지정해 줘야 한다. */
}

#nav {
    grid-area: nav;
}

#aside {
    grid-area: aside;
}

#section {
    grid-area: section;
}

#footer {
    grid-area: footer;
}

/* 반응형 (mediaquery사용) */
@media(max-width : 1220px) { /* 노트북 화면 */
    #wrap {
        width: 96%; /* 양쪽 여백 */
    }
}

@media(max-width : 768px) { /* 태블릿 화면 */
    #wrap {
        width: 100%; /* 여백없음 */
    }
}

@media(max-width : 480px) { /* 모바일 화면 */
    #wrap {
        grid-template-areas:  /* 영역 재정의 */
            "header header"
            "nav nav"
            "aside aside"
            "section section"
            "footer footer"
        ;
        grid-template-columns: auto;
        grid-template-rows: 100px 100px 380px 400px 100px;
    }
}

대표적인 Page LayOut 방식 3가지를 정리해 봤는데요, 이것말고도 많은 속성들을 이용하여 다양한 배치방법이 있지만, 사실 이 방법만으로 대부분의 레이아웃을 처리할 수 있습니다.
